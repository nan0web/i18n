# 🌱 SEED: @nan0web/i18n v1.5.0

> **Sovereign i18n Protocol**: Unified, Zero-LLM, Model-First architectural audit.

## 🚀 Що нового
- **Universal Inspector**: Нова команда `i18n inspect` для детермінованого аудиту архітектури i18n.
- **Zero-LLM Validation**: Аудит проводиться виключно через AST-парсер (Regex) — ніяких галюцинацій та витрат токенів.
- **Cross-package Support**: Параметризація шляхів (`--domain`, `--vocab`, `--ui`) дозволяє використовувати інструмент у будь-якому пакеті монорепозиторію.

## 📐 Стандарт "Model-as-Schema" (MaSaA)
Ми запроваджуємо єдиний стандарт перевірки цілісності локалізації:
1. **Source of Truth**: Ключі описуються виключно у статичних полях моделей (`static UI`, `static help`).
2. **Forbidden Hardcoding**: Використання літералів прямо у викликах `t('якась фраза')` в коді UI заборонено. Це ловиться інспектором.
3. **Vocab Integrity**: Автоматична перевірка наявності всіх ключів у словниках (`t.nan0`).

## 🛠 Використання
```bash
i18n inspect --domain=src/domain --vocab=play/data/uk/_/t.nan0 --ui=src/ui
```
